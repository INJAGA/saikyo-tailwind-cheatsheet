Svelte MCPサーバーを使用して、Svelte 5とSvelteKitの包括的なドキュメントにアクセスできます。以下のツールを効果的に活用してください。

## 利用可能なMCPツール

### 1. list-sections

最初に必ずこのツールを使用して、利用可能なドキュメントセクションをすべて確認してください。タイトル、use_cases、パスを含む構造化リストを返します。
SvelteまたはSvelteKitに関する質問を受けたら、関連セクションを見つけるために**必ずチャットの開始時にこのツールを使用**してください。

### 2. get-documentation

特定セクションの完全なドキュメントを取得します。単一または複数セクションを指定可能です。
list-sectionsツールを呼び出した後、返されたドキュメントセクション（特にuse_casesフィールド）を分析し、ユーザーのタスクに関連する**すべてのセクションをget-documentationで取得**してください。

### 3. svelte-autofixer

Svelteコードを分析して問題点と改善提案を返します。
Svelteコードを記述する際は、**ユーザーに送る前に必ずこのツールを使用**してください。問題や提案がなくなるまで繰り返し呼び出してください。

### 4. playground-link

提供されたコードでSvelte Playgroundリンクを生成します。
コード完成後、ユーザーにPlaygroundリンクが必要か確認してください。ユーザーの確認後にのみ呼び出し、プロジェクトのファイルにコードを書いた場合は**絶対に使用しない**でください。

## パッケージマネージャー

このプロジェクトでは**pnpm**を使用してください。`npm`や`yarn`は使用しないでください。

- パッケージのインストール: `pnpm add <package>`
- 開発用パッケージのインストール: `pnpm add -D <package>`
- スクリプト実行: `pnpm <script>`
- 開発サーバー起動: `pnpm run dev`

## スタイリングガイドライン

- **CSSは厳禁です。** 生のCSSや`<style>`ブロックは絶対に書かないでください。スタイリングにはTailwindCSSのユーティリティクラスのみを使用してください。
- **daisyUIコンポーネントを積極的に活用してください。** `btn`、`card`、`modal`、`badge`などのdaisyUIコンポーネントクラスを使ってコード量を削減し、一貫性を保ってください。
- **Iconifyを積極的に活用してください。** 直感性と使いやすさを向上させるために常にIconifyアイコンを使用してください。コンテキストに合った明確でわかりやすいアイコンを選んでください（例：`mdi:home`、`heroicons:trash`、`lucide:settings`）。
